# PITFALLS

- AutoLISP 兼容性：
  不要使用 Common Lisp 风格的可变列表函数，例如 `setcdr`、`setcar`、`push`、`pop`。提交前先扫一遍是否引入了 AutoLISP 不支持的函数。

- 外墙链尺寸识别：
  这类需求不能只按“穿过某一排线”来抓分隔线，否则会把更里面的墙线、柱边或墙厚一起标进去。优先按“贴着外侧基准墙线起止的墙面线”来取第一排链尺寸。

- 范围收口：
  用户已经把需求收窄到“沿第一版思路只修指定问题”时，不要顺手把基准线、第一排提取方式和标注层次一起重写。先锁住有效主逻辑，只改用户点名的偏移和阈值问题。

- 局部墙块端点：
  按局部墙块做第一排配对时，不能只认 span 中段，不认两头。很多该标的墙块封口线正好落在 span 端点附近；如果把端点排掉，就会只剩零散内部小段，用户框出来的整块反而漏标。

- 墙体候选来源：
  “最近内侧参考线”可以优先按白色墙线找，但侧边第一排局部墙块本身不能只从白色对象里取。实际图里很多该标的贴边端头和外墙边是绿色线；如果候选几何只收白色，第一排就会漏掉整段贴边墙块。

- Photoshop 图层样式同步：
  UXP 里用 `batchPlay` 写 `layerEffects` 时，字段名必须严格正确，例如 `RGBColor.green` 不能误写成别的键；同时不要只在“新建图层”那次才补样式。第一次写样式失败后，后续如果图层已存在却不再重试，就会出现用户一直看到“图层存在但没有 fx/内阴影”的假成功。

- Photoshop 灰地面与家具扣白：
  这套彩平里不要把“内部白块”一把梭分成同一类。文字、门扇、门洞弧线属于灰地面的一部分，默认应该跟着填灰；床、沙发、桌子、柜体这类内部闭合大白块才是需要单独扣回白的家具洞。只调一个面积阈值会同时做错这两边。

- Photoshop 家具白洞边界：
  家具白洞不能继续从“降采样后的灰区候选”里顺便扣。那样会只剩线白，而且边缘会因为 `dilate/close + nearest upscale` 超出线稿。家具扣白必须单独走原始线稿分辨率，再用原始白区边界回写。

- Photoshop 图层2 对照：
  用户给了正确 PSD 的 `图层2` 时，不准只看当前成图感觉对不对，更不准拿旧的中间合成图继续判断。先确认比较对象是“当前代码重新生成的最新图层2”，再和正确 `图层2` 做同口径对照；没有对照收敛前，不要把结果当成功版本往外报。

- Photoshop 图层2 `roomMask`：
  如果差异主要表现为“文字、门扇、细带区域没进灰里”，先检查分析分辨率是不是太低。`1200` 这档在这类彩平里会把不少细节直接吃掉；在确认是下采样丢细节前，不要一直在家具扣白规则上来回拧。

- CAD 裸数字触发插件功能：
  不要用 `:vlr-unknownCommand` 来接管用户直接输入的数字命令。它只能在 CAD 判定“未知命令”后触发，所以功能会生效，但命令行仍会提示“未知命令 100”，体验也慢。正确做法是注册真实的数字命令，例如动态定义 `C:1` 到 `C:999`，让 CAD 直接执行插件逻辑。触发关键词：`直接输数字`、`未知命令 100`、`unknownCommand`、`数字命令`。

- CAD MTEXT 房名视觉居中：
  现象是黄字/红字改字高后，设置中间中心对齐、再按 bbox 中心 `vla-Move` 仍然视觉偏。根因常见于房名是 `MTEXT`，内容里带 `\pi1707...;` 这类段落缩进码；bbox 或宽度盒子中心不等于文字视觉中心。已失败尝试是只改 `AttachmentPoint/InsertionPoint`，或改完字高后重新取 bbox 补偿移动。正确做法是先用 `entget` 确认对象是 `MTEXT`，清理小写 `\p...;` 段落格式、保留大写 `\P` 换行，再给段落写入 `\pxqc;` 居中，并用 MTEXT 原插入点作为中心。触发关键词：`MTEXT`、`\pi`、`\pxqc`、`bbox 中心移动无效`、`视觉居中仍偏`。

## 外墙链第一层局部墙块，不能继续按 `outer row + inner row + span` 去补

- 现象
  - “第一层外墙链尺寸漏标”连续两轮都卡在同一类问题：总尺寸正常，但第一层局部墙块会整段漏掉，或者只剩零散短段。
  - 在 `新块.dxf` 这类样本里，右侧目标图的第一层断点明显多于左侧当前结果；漏掉的往往不是整体外轮廓，而是每一侧从外往里看到的第一层局部墙块边界。

- 根因
  - 把第一层继续定义成“外墙厚度带里的一排短封口线”是错的。
  - `outer row + inner row + span` 这套模型，只能看见某一排线上直接可配对的短连接；它天然看不见通过白线、绿色贴边线、以及正交拐点连成的第一层局部墙块边界。
  - 目标图里的第一层，本质上更接近“每一侧从外侧看进去首先遇到的局部墙块/墙带边界投影”，不是“某一排线上的点两两配对”。
  - 后续样本又进一步证明：绿色线在这类图里常常不是墙体，而是窗线；但它们有时仍然会落在第一层链点的切分位置上。也就是说，绿色不能当“墙体 bodies”，却可能仍然是“角点/切分参考”。如果把绿色完全当墙，或者完全当空气，都会把第一层判错。
  - 真正难点不是“少一个函数”，而是“同一个靠角局部块到底归哪一侧第一层，以及绿色窗线只参与到哪一层切分”。这条归侧契约没定死，top/right/left/bottom 就会互相串边。

- 已失败尝试
  - 继续在 `outer row + inner row + span` 基础上补端点。
  - 把“桥接短封口线”从 `inner row` 改成 `outer row` 与 `inner row` 同时约束，仍然还是这条旧路。
  - 继续放宽或收紧墙厚、行容差，结果只是换一种漏法，没有把第一层对象定义改对。

- 正确做法
  - 先按“每一侧从外往里看到的第一层局部墙块/墙带边界”重新定义第一层，再决定代码实现。
  - 白线优先定义墙体本身；绿色线默认不要算成墙体。
  - 但不要把绿色线简单地整批丢掉。要先区分两件事：
    - 哪些对象是第一层墙体本身
    - 哪些对象只是第一层链点的切分参考
  - 靠角局部块必须先定“各归各边”的归侧规则，再谈具体阈值和 helper。
  - 如果发现思路又回到“先找 outer row，再找 inner row，再按 span 配对”，必须立即停，不要继续补阈值。

- 触发关键词
  - `第一层漏标`
  - `outer row`
  - `inner row`
  - `span`
  - `桥接短封口线`
  - `第一层局部墙块`
  - `绿色是窗户`
  - `各归各边`
