# Floor Fill Auto Handoff

## 当前目标口径

- 只先收 `图层 2` 和 `图层 7`
- `图层顺序`：
  - PDF 线稿最上
  - `图层 7`
  - `图层 2`
  - `PDF白底`
- `图层 2` 现阶段只先做：
  - 整页背景灰
  - 主户型内浅灰地面
  - 家具内部整块白洞
- `暂不处理`：
  - 更深装饰灰块
  - 蓝灰块

## 这次已经落下的改动

- `main.js`
  - 分析分辨率从 `1200` 提到 `1800`
  - `roomMask` 上采样回原图后，额外做一次 `closeMask(..., 1)`
  - 自动补 `PDF白底`
  - 每次运行都强制同步 `图层 7` 的默认 `Inner Shadow`

- `region-engine.js`
  - `roomMask` 走主户型范围识别
  - 家具扣白单独走原始线稿分辨率
  - 家具候选已经加了一批假白洞过滤，重点压：
    - 靠边的大空房间口袋
    - 条带/点阵类误判
    - 小服务区里的细长假白块

## 已确认的调试结论

- 不要再把主要精力放在“继续拧家具扣白阈值”上。
- 这轮对照后已经确认：
  - `1200` 分析高度会直接吃掉很多文字、门扇、细带细节
  - 这类差异的大头，其实来自 `roomMask` 太粗，不是家具白洞本身
- 所以这次已经把主方向改成：
  - 提高分析分辨率
  - 在原图尺度上轻量补 `roomMask`

## 当前状态

- 比前一版更接近正确 `图层 2`
- 家具内部白块已经基本成型，不再只是线白
- 典型假白洞已经压掉一批
- 但还没有完全收口，仍然不能当最终成功版

## 继续接力时的建议顺序

1. 先用当前代码重新生成最新 `图层 2`
2. 必须对照正确 PSD 里的最新 `图层 2`
3. 如果差异主要是：
   - 文字 / 门扇 / 细带没进灰
   - 先看 `roomMask`
4. 如果差异主要是：
   - 床 / 沙发 / 桌子没有整块白
   - 再看家具扣白
5. 在你开始做更深灰 / 蓝灰之前，先把当前浅灰 / 白洞口径收稳
